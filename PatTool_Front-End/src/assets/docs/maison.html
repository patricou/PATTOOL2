Accès

La plupart des entrées du menu Maison nécessitent le rôle Keycloak Iot. Exception : Scan sécurité (mandat), visible pour tout utilisateur connecté.

Réseau local est réservé aux administrateurs.

Domotique

Route : #/iot

  • Test Ethernet Shield — vérifie la liaison avec le microcontrôleur domotique
  • Portail — ouverture / fermeture du portail (commande distante via le serveur PatTool)
  • Volets / stores — commandes par zone si configurées côté serveur
  • États et réponses affichés dans des cartes avec indicateur visuel
Les commandes passent par le backend PatTool qui dialogue avec le matériel sur le réseau local — pas de contrôle direct depuis le navigateur.

Réseau local

Route : #/iot/local-network — admin uniquement

  • Scan des appareils sur le LAN (adresses IP, noms, ports ouverts selon configuration)
  • Tableau filtrable et export des résultats
  • Utile pour diagnostiquer la connectivité domotique et caméras

Caméras

Route : #/iot/cameras

  • Flux vidéo des caméras configurées sur le serveur (proxy MJPEG ou snapshot)
  • Grille multi-caméras, rechargement manuel
  • Accès protégé par authentification et rôle IoT

Proxies LAN

Route : #/iot/proxy

  • Liste des proxys HTTP configurés pour atteindre des services internes (Arduino, caméras, etc.)
  • Test de connectivité par proxy
  • Documentation des URLs relais utilisées par le backend

Scan sécurité (mandat)

Route : #/tools/security-scan — tout utilisateur connecté (sans rôle IoT)

  • Outil de sensibilisation : liens vers ressources officielles (ANSSI, CVE, bonnes pratiques)
  • Périmètre d'évaluation documenté — pas un scan actif de votre machine
  • Accessible depuis Maison pour rappeler les enjeux de sécurité des objets connectés

Access

Most Home menu entries require Keycloak role Iot. Exception: Security scan (mandate) for any logged-in user.

Local network is admin-only.

Home automation

Route: #/iot — test Ethernet shield, gate open/close, shutters; commands via PatTool server.

Local network

Route: #/iot/local-network — admin LAN device scan and diagnostics.

Cameras

Route: #/iot/cameras — configured camera streams (MJPEG/snapshot grid).

LAN proxies

Route: #/iot/proxy — HTTP proxies to internal services, connectivity tests.

Security scan (mandate)

Route: #/tools/security-scan — awareness links and documented assessment scope (no active machine scan).
