1. File System Photo Selection Feature

Feature Overview: Added directory selection functionality for "Photo from File System" link type in event creation component, matching the existing functionality in the update component.

1.1 Description

When creating a new event, users can now add a link of type "Photo from File System" (PHOTOFROMFS) and select a directory from their local file system. This feature was previously only available in the event update component and has now been extended to the creation component.

1.2 User Interface Changes

The following UI elements were added to the event creation form:

  • Directory Selection Button: A folder icon button appears next to the link input field when "Photo from File System" is selected as the link type
  • File Input Field: A hidden file input with webkitdirectory attribute for directory selection
  • Input Type Change: The link input field type was changed from url to text to accommodate file system paths

1.3 Technical Implementation

Frontend Components Modified:

  • Frontend create-evenement.component.html
  • Frontend create-evenement.component.ts

Key Methods Added:

3. Technical Implementation Details

3.1 File System Access API

The directory selection feature uses the modern File System Access API when available, providing a native file picker experience. For browsers that don't support this API, the implementation falls back to the traditional webkitdirectory file input.

Browser Compatibility

  • Modern Browsers: Chrome 86+, Edge 86+ - Uses File System Access API
  • Older Browsers: Firefox, Safari, older Chrome - Uses webkitdirectory fallback

3.2 Year Prefix Logic

For PHOTOFROMFS type links, the system automatically detects if a directory name starts with a 4-digit year (YYYY format, between 1900-2100) and adds it as a prefix if not already present. This ensures consistent path formatting for file system photo directories.

private addYearPrefixIfNeeded(link: string): string { // Checks if first 4 chars are a valid year (YYYY) // Adds "YYYY/" prefix if needed // Handles duplicate prefix prevention }

3.3 Path Normalization

The implementation includes comprehensive path normalization to handle differences between Windows and Unix-like systems:

  • Converts backslashes to forward slashes (and vice versa for Windows)
  • Handles UNC paths (Windows network paths)
  • Removes duplicate slashes
  • Preserves platform-specific path formats

3.4 Backend Search Integration

The backend search functionality has been extended to recognize wine-related keywords in multiple languages. When users search for events, the system will match events of type "Wine" using any of the registered keywords.

Note: The search is case-insensitive and supports partial matches, making it easy to find wine-related events regardless of the language used in the search query.
